# Terminal File Input

This note documents terminal drag/drop and paste-to-path behavior.

## Goals

- Dragging files onto an active terminal inserts quoted file paths.
- Pasting copied files from Windows Explorer inserts the original file paths.
- Pasting image-only clipboard content, such as a WeChat screenshot, saves a
  temporary image file and inserts that path.
- Right-click paste uses the same copied-file and image handling before falling
  back to plain text clipboard content.
- Cleaning temporary files only removes files generated by paste handling. It
  never deletes original files copied from the system.

## Frontend Flow

Terminal file input is split between App-level drag/drop and terminal-level
paste handling.

- `src/App.tsx` registers one Tauri drag/drop listener with
  `getCurrentWebview().onDragDropEvent`.
- Drop handling only writes to the active terminal and only if the physical
  drop position, converted to CSS pixels, lands inside that terminal's xterm
  element.
- `src/components/TerminalView.tsx` registers paste handling in capture phase on
  the terminal container so it can run before xterm's helper textarea consumes
  the event.
- `TerminalView.tsx` also owns a small terminal context menu with only `复制`
  and `粘贴`. The paste action first calls `saveSystemClipboardFiles()` so
  copied files and screenshots become shell-quoted paths. If no paths are
  available, it reads regular text and routes it through
  `src/utils/pasteManager.ts`.
- `pasteManager.ts` writes text directly to the PTY. For bracketed paste, it
  normalizes pasted line breaks to LF and trims trailing line breaks before
  adding explicit bracketed paste markers. This avoids turning pasted CRLF
  lines into submit-style Enter input in raw-mode TUIs such as Codex.
- `src/utils/terminalPaths.ts` owns shell-aware path formatting used by both
  drag/drop and paste.
- A single drag/drop or copied-file paste inserts at most 10 paths. Extra paths
  are ignored.
- The frontend prevents browser/xterm paste handling for terminal paste events,
  checks `saveSystemClipboardFiles()` first, and only falls back to text when no
  file or screenshot paths are available. Keyboard paste and right-click paste
  share this path.

Path insertion is shell-aware:

- PowerShell and PowerShell 7 use single-quoted Windows paths.
- CMD uses double-quoted Windows paths.
- Git Bash converts backslashes to slashes and uses POSIX quoting.
- WSL converts drive paths such as `C:\Users\a.png` to `/mnt/c/Users/a.png`
  and uses POSIX quoting.

## Backend Flow

`src-tauri/src/lib.rs` exposes:

- `save_system_clipboard_files() -> Vec<String>`
- `clear_paste_temp_files() -> usize`

On Windows, `save_system_clipboard_files` opens the system clipboard, copies the
needed clipboard data, closes the clipboard, and only then writes generated temp
files to disk. This keeps clipboard ownership short.

It handles three cases in order:

1. `CF_HDROP` file list
   - Used when files are copied from Explorer.
   - Returns the original paths.
   - Does not copy or cache these files.
2. Registered image formats
   - Used for image-only clipboard content like screenshots.
   - Supported formats include `image/png`, `PNG`, `image/jpeg`, `image/jpg`,
     `image/bmp`, and `image/webp`.
   - Rejects image payloads larger than 20 MB before copying them into process
     memory.
   - Saves the image bytes under `%TEMP%\mutishell\paste-temp`.
   - Returns the generated temporary path.
3. Windows bitmap clipboard formats
   - Falls back to `CF_DIB` and `CF_DIBV5` when compressed image formats are not
     available.
   - Rejects bitmap payloads larger than 20 MB before converting them to BMP.
   - Wraps DIB bytes in a BMP file header and saves a `.bmp` file under
     `%TEMP%\mutishell\paste-temp`.

The clipboard open operation retries briefly because screenshot tools and other
apps may hold the clipboard immediately after copying.

## Temporary Files

Only clipboard content without an original path creates temporary files. Current
examples are screenshot/image clipboard formats.

Generated files are written to:

```text
%TEMP%\mutishell\paste-temp
```

The topbar cleanup button is labeled `清理临时文件`. It calls
`clear_paste_temp_files` and removes only supported generated image files from
that directory. It does not delete files that were copied from Explorer.

## Verification

Run:

```powershell
npm run build
cd src-tauri
cargo check
```

Manual checks:

- Drag a local file onto PowerShell and confirm a quoted path is inserted.
- Drag more than 10 files and confirm only the first 10 paths are inserted.
- Copy a PDF from Explorer and paste into the terminal; the original PDF path
  should be inserted.
- Copy a PDF from Explorer, right-click the terminal, choose `粘贴`, and confirm
  the original PDF path is inserted.
- Copy multiline text and paste into an interactive TUI such as Codex; it should
  arrive as one paste instead of submitting at the first newline.
- Copy a WeChat screenshot and paste into the terminal; a path under
  `%TEMP%\mutishell\paste-temp` should be inserted.
- Copy a WeChat screenshot, right-click the terminal, choose `粘贴`, and confirm
  a path under `%TEMP%\mutishell\paste-temp` is inserted.
- Select terminal text, right-click the terminal, choose `复制`, and confirm the
  selected text is copied.
- Click `清理临时文件`; generated screenshot files should be removed, while
  original copied files remain untouched.
